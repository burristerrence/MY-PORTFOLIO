# Step-by-Step: Create GitHub Repository

## 🎯 Goal
Create a new repository on GitHub to store your portfolio code.

---

## 📋 Step-by-Step Instructions

### Step 1: Go to GitHub
1. Open your web browser
2. Navigate to **https://github.com**
3. **Sign in** to your GitHub account
   - If you don't have an account, click "Sign up" and create one first

### Step 2: Create New Repository
1. Look at the **top-right corner** of the GitHub page
2. Click the **"+" icon** (plus sign)
3. From the dropdown menu, select **"New repository"**

### Step 3: Fill in Repository Details

#### Repository Name
- **Name**: `portfolio` (or `my-portfolio`, `terrence-burris-portfolio`)
- Keep it simple and descriptive
- Use lowercase letters, numbers, and hyphens only

#### Description (Optional but Recommended)
```
Professional portfolio website built with Next.js 14, TypeScript, and Tailwind CSS
```

#### Visibility
- Choose **Public** (recommended) - Free and allows you to deploy easily
- Or **Private** - If you want to keep it private

#### ⚠️ IMPORTANT: Do NOT Check These Boxes!
- ❌ **DO NOT** check "Add a README file" (we already have one)
- ❌ **DO NOT** check "Add .gitignore" (we already have one)
- ❌ **DO NOT** check "Choose a license" (optional, but not needed now)

**Why?** Because we already have these files in our local repository, and checking these boxes will create conflicts.

### Step 4: Create the Repository
1. Click the green **"Create repository"** button
2. GitHub will show you a page with setup instructions
3. **Don't follow those instructions yet!** We'll do it differently since we already have code

---

## ✅ What You Should See

After creating the repository, you'll see:
- A page with your repository URL (like `https://github.com/YOUR_USERNAME/portfolio`)
- Some setup instructions (we'll ignore the "create new repo" part)
- An empty repository (or with a README if you accidentally checked that box)

---

## 📝 Information You'll Need

After creating the repository, you'll need:
1. **Your GitHub username** (from the URL)
2. **Repository name** (what you named it)
3. **Repository URL** (shown on the page)

**Example:**
- Username: `terrenceburris`
- Repository: `portfolio`
- URL: `https://github.com/terrenceburris/portfolio.git`

---

## 🚀 Next Steps (After Creating Repository)

Once you've created the repository, come back here and I'll help you:
1. Connect your local repository to GitHub
2. Push your code to GitHub
3. Verify everything is uploaded correctly

---

## 💡 Tips

- **Repository name**: Keep it simple and memorable
- **Public vs Private**: Public repositories are free and great for portfolios
- **Description**: Helps others (and future you) understand what the project is

---

## ❓ Troubleshooting

### "Repository name already exists"
- Try a different name like `portfolio-website` or `my-portfolio-site`
- Or add your name: `terrence-burris-portfolio`

### "I accidentally checked 'Add README'"
- That's okay! We can handle it when we push
- Just let me know and I'll help you resolve any conflicts
